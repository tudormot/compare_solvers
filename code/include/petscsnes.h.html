175: /*E
176:     SNESConvergedReason - reason a SNES method was said to
177:          have converged or diverged

179:    Level: beginner

181:    The two most common reasons for divergence are
182: $   1) an incorrectly coded or computed Jacobian or
183: $   2) failure or lack of convergence in the linear system (in this case we recommend
184: $      testing with -pc_type lu to eliminate the linear solver as the cause of the problem).

186:    Diverged Reasons:
187: .    SNES_DIVERGED_LOCAL_MIN - this can only occur when using the line-search variant of SNES.
188:        The line search wants to minimize Q(alpha) = 1/2 || F(x + alpha s) ||^2_2  this occurs
189:        at Q'(alpha) = s^T F'(x+alpha s)^T F(x+alpha s) = 0. If s is the Newton direction - F'(x)^(-1)F(x) then
190:        you get Q'(alpha) = -F(x)^T F'(x)^(-1)^T F'(x+alpha s)F(x+alpha s); when alpha = 0
191:        Q'(0) = - ||F(x)||^2_2 which is always NEGATIVE if F'(x) is invertible. This means the Newton
192:        direction is a descent direction and the line search should succeed if alpha is small enough.

194:        If F'(x) is NOT invertible AND F'(x)^T F(x) = 0 then Q'(0) = 0 and the Newton direction
195:        is NOT a descent direction so the line search will fail. All one can do at this point
196:        is change the initial guess and try again.

198:        An alternative explanation: Newton's method can be regarded as replacing the function with
199:        its linear approximation and minimizing the 2-norm of that. That is F(x+s) approx F(x) + F'(x)s
200:        so we minimize || F(x) + F'(x) s ||^2_2; do this using Least Squares. If F'(x) is invertible then
201:        s = - F'(x)^(-1)F(x) otherwise F'(x)^T F'(x) s = -F'(x)^T F(x). If F'(x)^T F(x) is NOT zero then there
202:        exists a nontrival (that is F'(x)s != 0) solution to the equation and this direction is
203:        s = - [F'(x)^T F'(x)]^(-1) F'(x)^T F(x) so Q'(0) = - F(x)^T F'(x) [F'(x)^T F'(x)]^(-T) F'(x)^T F(x)
204:        = - (F'(x)^T F(x)) [F'(x)^T F'(x)]^(-T) (F'(x)^T F(x)). Since we are assuming (F'(x)^T F(x)) != 0
205:        and F'(x)^T F'(x) has no negative eigenvalues Q'(0) < 0 so s is a descent direction and the line
206:        search should succeed for small enough alpha.

208:        Note that this RARELY happens in practice. Far more likely the linear system is not being solved
209:        (well enough?) or the Jacobian is wrong.

211:    SNES_DIVERGED_MAX_IT means that the solver reached the maximum number of iterations without satisfying any
212:    convergence criteria. SNES_CONVERGED_ITS means that SNESConvergedSkip() was chosen as the convergence test;
213:    thus the usual convergence criteria have not been checked and may or may not be satisfied.

215:    Developer Notes: this must match petsc/finclude/petscsnes.h

217:        The string versions of these are in SNESConvergedReasons, if you change any value here you must
218:      also adjust that array.

220:    Each reason has its own manual page.

222: .seealso: SNESSolve(), SNESGetConvergedReason(), KSPConvergedReason, SNESSetConvergenceTest()
223: E*/

226:               SNES_CONVERGED_FNORM_RELATIVE    =  3, /* ||F|| < rtol*||F_initial|| */
227:               SNES_CONVERGED_SNORM_RELATIVE    =  4, /* Newton computed step size small; || delta x || < stol || x ||*/
228:               SNES_CONVERGED_ITS               =  5, /* maximum iterations reached */
229:               SNES_CONVERGED_TR_DELTA          =  7,

232:               SNES_DIVERGED_FUNCTION_COUNT      = -2,
233:               SNES_DIVERGED_LINEAR_SOLVE        = -3, /* the linear solve failed */

392: /*MC
393:     SNES_NORM_NONE - Don't compute function and its L2 norm.

395:    Level: advanced

397:     Notes:
398:     This is most useful for stationary solvers with a fixed number of iterations used as smoothers.

400: .seealso: SNESNormSchedule, SNESSetNormSchedule(), SNES_NORM_DEFAULT
401: M*/

403: /*MC
404:     SNES_NORM_ALWAYS - Compute the function and its L2 norm at each iteration.

406:    Level: advanced

408:     Notes:
409:     Most solvers will use this no matter what norm type is passed to them.

411: .seealso: SNESNormSchedule, SNESSetNormSchedule(), SNES_NORM_NONE
412: M*/

414: /*MC
415:     SNES_NORM_INITIAL_ONLY - Compute the function and its L2 at iteration 0, but do not update it.

417:    Level: advanced

419:    Notes:
420:    This method is useful in composed methods, when a true solution might actually be found before SNESSolve() is called.
421:    This option enables the solve to abort on the zeroth iteration if this is the case.

423:    For solvers that require the computation of the L2 norm of the function as part of the method, this merely cancels
424:    the norm computation at the last iteration (if possible).

426: .seealso: SNESNormSchedule, SNESSetNormSchedule(), SNES_NORM_FINAL_ONLY, SNES_NORM_INITIAL_FINAL_ONLY
427: M*/

429: /*MC
430:     SNES_NORM_FINAL_ONLY - Compute the function and its L2 norm on only the final iteration.

432:    Level: advanced

434:    Notes:
435:    For solvers that require the computation of the L2 norm of the function as part of the method, behaves
436:    exactly as SNES_NORM_DEFAULT.  This method is useful when the function is gotten after SNESSolve and
437:    used in subsequent computation for methods that do not need the norm computed during the rest of the
438:    solution procedure.

440: .seealso: SNESNormSchedule, SNESSetNormSchedule(), SNES_NORM_INITIAL_ONLY, SNES_NORM_INITIAL_FINAL_ONLY
441: M*/

443: /*MC
444:     SNES_NORM_INITIAL_FINAL_ONLY - Compute the function and its L2 norm on only the initial and final iterations.

446:    Level: advanced

448:    Notes:
449:    This method combines the benefits of SNES_NORM_INITIAL_ONLY and SNES_NORM_FINAL_ONLY.
